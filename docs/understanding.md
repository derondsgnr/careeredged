# My Understanding

## The Mission
Full UI/UX overhaul of a large product. Backend (schema, APIs) stays untouched unless absolutely necessary. The existing UI was AI-generated and lacks craft, intentionality, and delight. My job is to bring all of that.

## The Product
A career-oriented platform ("GPS for career success") with significant complexity: ~6 user types, 20+ features. The #1 structural problem is **navigation** -- how to make a dense, multi-persona system feel simple, intuitive, and not overwhelming. This is a systems design problem first, visual design problem second.

## Creative Direction (from your uploads)
- **Palette:** Cyan (#22D3EE), Lime/Neon Green (#B3FF3B), Deep Forest Green (#042C01), Near-Black (#1A1A1B). High contrast, bold, energetic but grounded. The dark base with electric accents says "modern, forward-moving, intelligent."
- **Motifs:** Compass symbols, directional markers, pathfinding elements, structured navigation graphics. This isn't decorative -- it's functional metaphor. The brand *is* guidance.
- **Brand Feeling:** Guiding, intelligent, reassuring. Not cold-techy, not corporate-warm. The sweet spot of a tool that knows what it's doing and makes you feel like you do too.
- **Typography:** Onest or Urbanist (headings) + Satoshi (body), replacing FT Sterling. I'll study our north star products' type systems for scale, hierarchy, and rhythm before committing to the pairing.
- **Logo direction:** Bold geometric marks with directional energy -- arrows, chevrons, forward motion. The brand assets shown (Clickoy, Actnova, Pickup, etc.) all share this DNA.

## North Star Products
Apple, Linear, Airbnb, Vercel, Claude, Resend. These set the bar for:
- **Apple:** Restraint, hierarchy, making complex feel simple, emotional resonance
- **Linear:** Speed, density done right, keyboard-first UX, dark mode mastery
- **Airbnb:** Warmth in function, progressive disclosure, trust-building UI
- **Vercel:** Developer-grade clarity, beautiful minimalism, information architecture
- **Claude:** Conversational intelligence, calm UI, thoughtful empty states
- **Resend:** Clean developer tools, beautiful email/API UX, tasteful restraint
- **Firecrawl:** Developer tool clarity, light-theme craft, functional simplicity, usage visualization
- **Manus:** AI agent UX — task state visibility, empty state warmth, progressive disclosure of complexity
- **Flora:** Spatial/canvas-based interaction, block-as-unit paradigm, dark canvas mastery, contextual transformation menus
- **Weavy:** Node-based workflow with credit economy visibility, model selection grids, configuration panels
- **ClientIQ:** Light-theme dashboard density, KPI card hierarchy, green accent on light backgrounds, conversion funnel visualization

Also draw inspiration from how they handle information architecture and hierarchy across these products

## How I'll Analyze Your 30+ References
For each reference, I'll obsessively study:
- **Why it works** -- the underlying UX logic, not just aesthetics
- **Surface treatment** -- shadows, spacing, gradients, glass, patterns, background treatments on every surface type (buttons, cards, charts, backgrounds)
- **Component choices** -- why certain data types got certain UI patterns
- **Layout & composition** -- the grid logic, rhythm, breathing room
- **Typography** -- scale, weight distribution, how type creates hierarchy
- **Emotional design** -- what feeling were they engineering, and how
- **Design language fit** -- how the visual system serves the product's purpose
- **Abstractions** -- reusable patterns and systems we can extract and apply

Also study the hierarchy, information architecture

## Our Standards
- Visual execution must match and surpass references + north stars

- Thoughtful, careful, tasteful affordances
- Deep user understanding driving every decision
- Delight, seamlessness, intuitiveness, simplicity

## Working Principles

- **No assumptions.** Don't create things that aren't there, can't be built, or don't fit the system.
- **Context first.** Ask questions to ensure full understanding before acting.
